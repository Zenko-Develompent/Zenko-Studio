# Условные операторы в Bash: практика

Здесь тренируем задачи на ветвление.

## Шаблон

```bash
read n
if (( n > 0 )); then
  echo "YES"
else
  echo "NO"
fi
```

## Полезные конструкции

- Проверка диапазона: `(( n >= l && n <= r ))`
- Цепочка категорий: `if / elif / else`

## Частые ошибки

- Потерян `fi`.
- Неправильный вид сравнения.
- Не проверены граничные случаи (`0`, отрицательные).

## Рекомендация

Прогоняй решение вручную на 3-4 тестах до отправки.

Главная цель: писать точные и предсказуемые условия.
